# Project Plan

## Risk Analysis (TAME Framework)
1. **Risk:** Tight deadline with multiple components to complete / scope creep
   - **Mitigate** - Adhere to detailed project scope and work breakdown structure
2. **Risk:** Technical issues with CSS styling or deployment on GitHub Pages
   - **Mitigate** - Utilize online resources, seek help from instructor/peers if necessary, and allow time for early testing and troubleshooting
3. **Risk:** Missing required files or components in final submission
   - **Eliminate** - Use a checklist based on project requirements and rubric to ensure all components are included before submission

## Work Breakdown Structure (WBS)
1. **Planning** (1-2 hours)
   - Define project scope and goals
   - Create project plan and schedule
   - Perform regular retrospectives and adjustments as needed
2. **Website Development** (3-5 hours)
   - Create index.html with semantic structure and content
   - Create about.html with relevant information and styling
   - Develop style.css for consistent design and responsiveness
3. **Documentation & Polish** (1-2 hours)
   - Write README.md with project description, features, and instructions
   - Ensure proper file structure and organization
   - Test website on GitHub Pages and fix any issues
4. **Final Review & Submission** (1 hour)
   - Review project against rubric and checklist
   - Submit project and deploy on GitHub Pages
